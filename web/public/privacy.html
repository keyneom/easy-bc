EasyBC privacy policy

Effective June 21, 2026

EasyBC is local-first. It has no EasyBC account server, advertising SDK, or analytics SDK.

Data stored locally

Period records, day logs, body-signal observations, planner settings, and generated plans stay in application or browser storage. Android platform backup is disabled.

Optional encrypted cloud sync

When explicitly enabled in the web or Android app, EasyBC requests only the Google Drive app-data scope. It encrypts user-authored records on the device before placing one encrypted file in Drive's hidden application-data folder. Raw passkey output is kept only in memory for the active operation. A short-lived Google access token and the derived content-encryption key may remain in tab or Android-process memory for automatic sync, but are never intentionally written to browser or app storage and are cleared when the tab/process ends or after 15 minutes in the background.

EasyBC uses Google user data only to provide encrypted cloud sync requested by the user and does not sell or transfer that data to third parties.

User-directed exports

Manual JSON backups, iCalendar exports, and entries written to a device calendar are not encrypted by EasyBC. Treat them as sensitive files.

Retention and deletion

Local data remains until records or application storage are deleted. Encrypted cloud data remains in the Drive app-data folder until the user chooses Delete encrypted cloud copy in EasyBC or removes EasyBC's hidden app data through their Google Account. Revoking access prevents future access but does not by itself guarantee file deletion.

Security reports

Use the repository's private security advisory form. Do not post personal health data in public issues.
